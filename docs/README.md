# Documentation

This directory contains completion reports, QA results, audit reports, and implementation guides generated by ECHO v1.4.0's auto-audit system.

## File Naming Convention

All documentation files follow this format:
- `COMPLETION_REPORT_[FID]_[YYYYMMDD].md` - Feature implementation summaries
- `QA_RESULTS_[FID]_[YYYYMMDD].md` - Testing and validation reports
- `AUDIT_REPORT_[TYPE]_[YYYYMMDD].md` - Quality control audit results
- `IMPLEMENTATION_GUIDE_[FID]_[YYYYMMDD].md` - Technical implementation documentation
- `SUMMARY_[PHASE/SPRINT]_[YYYYMMDD].md` - Phase or sprint completion summaries

## Purpose

These files are automatically generated by `GENERATE_DOCUMENTATION()` function when features are completed. They provide:
- Comprehensive implementation summaries
- Quality metrics and verification results
- Lessons learned and best practices
- Technical documentation for future reference

## Auto-Archiving

Documentation files are automatically linked from `dev/completed.md` entries and archived with completed features when `AUTO_ARCHIVE_CHECK()` executes.
